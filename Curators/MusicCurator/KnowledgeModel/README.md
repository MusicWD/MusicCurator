# KnowledgeModel

The KnowledgeModel is the conceptual and epistemic foundation of the project.

It is not ordinary documentation. It defines the language in which the system thinks.

## Subfolders

- `Foundations/` – Fundamental model concepts and principles.
- `Vocabulary/` – Controlled vocabularies.
- `Decisions/` – Architecture Decision Records.
- `Examples/` – Worked examples.
- `Diagrams/` – Visual models and diagrams.

## Rule

No implementation should introduce model concepts that are not grounded here.
