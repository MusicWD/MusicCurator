# 010 – Design Principles

Status: Draft  
Version: 0.1

## Principle 1: The model does not force assertions

An entity may exist without assertions about every possible property.

## Principle 2: Absence of assertion is not assertion

Missing data does not mean:
- unknown
- false
- not applicable
- not existing

It means only: no assertion has been made.

## Principle 3: Every assertion has context

Every assertion should be able to carry:
- Agent
- Evidence
- Scope
- Confidence
- Validity
- Source

## Principle 4: Relations create context, not truth

A relation between entities does not automatically generate further assertions.

## Principle 5: Containment is not inheritance

Structural membership never implies semantic inheritance.
